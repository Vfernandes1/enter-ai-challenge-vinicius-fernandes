# Skill: Diretrizes de Formatação e Correção do Relatório Final

## Erros Históricos a Serem Evitados (Anti-Patterns)
Baseado nas inconsistências da versão V1 do nosso MVP, aplique as seguintes correções estritas:

1. **Erro de Identificação:** O relatório anterior saudava o cliente como "Prezado João"[cite: 4].
   * **Regra:** Utilize sempre o nome correto extraído dos dados do portfólio (Prezado Albert).
2. **Alucinação de Cenário Macro:** O relatório anterior citou que o Federal Reserve iniciaria cortes de juros em julho e que a Selic se estabilizaria em 9%[cite: 4].
   * **Regra:** Estes dados estão incorretos e conflitam com o relatório oficial da XP. A Selic terminal projetada é de 15,50% e não há previsão de cortes nos EUA neste ano[cite: 1]. Use apenas os dados da `skill_macro_analysis.md`.
3. **Erros de Previsão Cambial:** O relatório anterior previu o Dólar a R$ 4,70 para o fim de 2024[cite: 4]. 
   * **Regra:** Atualize para as projeções oficiais mais recentes: R$ 6,20 para 2025[cite: 1].

## Padrões de Geração de Texto
* **Idioma e Formato:** A carta mensal a ser compartilhada com o cliente deve estar em formato de carta (letter format), em português.
* **Tamanho Máximo:** O relatório mensal deve ter no máximo duas páginas.
* **Tom de Voz:** Profissional, empático, consultivo e traduzido para termos leigos. Assine a carta com o nome do assessor: Antonio Bicudo[cite: 3, 4].
